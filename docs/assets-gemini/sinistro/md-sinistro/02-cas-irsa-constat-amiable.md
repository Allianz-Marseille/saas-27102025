## 1. Contexte IRSA / IDA

- **IRSA** (Indemnisation Directe et de Recours entre Sociétés d’Assurance) est la convention cadre qui gère les accidents **matériels** entre véhicules. Elle définit les **Cas** (10, 13, 21…) pour répartir les responsabilités entre conducteurs A et B.
- **IDA** (Indemnisation Directe de l’Assuré) est le **volet opérationnel** de l’IRSA : chaque assureur indemnise **directement** son propre client, sans attendre le recours. Le constat amiable (papier ou **e-constat**) déclenche l’application de ces règles.
- Le constat a la même valeur juridique qu’il soit rempli sur papier ou via l’application **e-constat auto** (DARVA).

## 3. Lecture du constat (A/B, croix, schéma)

Champs utiles pour le bot lors de l’analyse d’un constat (papier ou e-constat) :

- **Véhicule A / Véhicule B** : identification des deux parties (assureur, contrat, immatriculation).
- **Croix (cases)** : les cases cochées décrivent les circonstances (ex. « Mon véhicule était à l’arrêt », « Je circulais en marche arrière », etc.) ; elles permettent de faire correspondre la situation à un **Cas** IRSA.
- **Schéma** : croquis de l’accident (position des véhicules, sens de circulation) ; aide à valider ou affiner le cas retenu.
- **Lieu, date, heure, témoins** : pour traçabilité et recours.

---

## 4. Assureur gestionnaire et recours

- **Assureur gestionnaire** : dans le cadre IDA, chaque assureur est gestionnaire pour **son** assuré ; il indemnise directement son client puis règle le recours avec l’autre assureur selon le cas IRSA (répartition de responsabilité).
- **Recours** : une fois le cas déterminé (ex. 100 % A, 0 % B), l’assureur du conducteur B (non responsable) exerce un recours contre l’assureur du conducteur A (responsable) pour récupérer les sommes versées à son client.
